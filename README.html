Security Features

Every allocation is protected by multiple detection layers

🛡️
Buffer Overflow Detection
Canary values placed before and after every allocation. Any write beyond the boundary is caught immediately before it can cause damage.
CANARY VALUES
☠️
Use-After-Free Detection
Freed memory is poisoned with 0xDD pattern. Any attempt to read or write freed memory is detected at the byte level.
POISON MEMORY
🔁
Double Free Detection
Every block tracks its free status. Calling free twice on the same pointer is caught and blocked before heap corruption occurs.
STATE TRACKING
💧
Memory Leak Detection
Every allocation records its source file and exact line number. At any point you can get a full report of every byte that was never freed.
FILE + LINE TRACKING
🪝
Malloc Hook
Overrides malloc, free, calloc and realloc at the compiler level. Any existing C program is automatically protected without changing a single line of code.
ZERO CODE CHANGE
📊
Live Dashboard
C# dashboard reads the SQLite database in real time showing allocation history, severity levels, ASCII graphs and CSV export.
C# + SQLITE

How It Works

Drop-in replacement — zero changes to existing code

example.c — standard code, fully protected
#include <stdlib.h> int main() { /* Standard malloc — intercepted by SecureMalloc */ int* data = (int*)malloc(sizeof(int) * 10); data[0] = 42; /* SecureMalloc catches this automatically: */ free(data); free(data); // DANGER: Double free detected! return 0; }
output — SecureMalloc catches the exploit
[SECUREMALLOC] Allocated 40 bytes at 0x000001948A8027E8 [SECUREMALLOC] Freed block at 0x000001948A8027E8 [SECUREMALLOC] DANGER: Double free detected at 0x000001948A8027E8!
